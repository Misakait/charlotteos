# Charlotte OS

Charlotte OS 是一个面向 `riscv64` 平台的个人 Rust 操作系统内核项目，采用 `no_std` 方式开发，并在 QEMU `virt` 机器和 RustSBI 环境下进行测试。当前项目已经实现了内核早期启动、内存管理、任务调度、异常与中断处理、UART 控制台输出，以及基础的系统调用支持。

需要特别说明的是：**这是我的个人项目，不是课程实验**。我会把它当作一个持续演进的系统软件工程项目来维护和完善。

## 项目简介

这个项目的目标不是简单“跑起来一个内核”，而是围绕一个可逐步扩展的 RISC-V 操作系统内核，持续构建如下能力：

- 从汇编入口进入 Rust 内核
- 建立早期页表并切换到高半区虚拟地址空间
- 解析设备树，识别内存与保留区域
- 初始化物理内存管理与页表映射
- 建立任务调度器并运行多个内核任务
- 处理定时器中断和外部中断
- 提供基于 UART 的输出和基础输入支持
- 为后续用户态、文件系统、更多系统调用等模块预留扩展空间

目前代码中包含一些用于验证调度器和 I/O 路径的演示任务，例如 `test_task_a`、`test_task_b` 和一个简单的 `shell` 任务。它们的作用是帮助我验证抢占、睡眠、唤醒、退出以及任务切换的逻辑是否正常工作。

## 设计目标

Charlotte OS 目前的设计更偏向“可演进的个人内核原型”，主要关注以下几点：

1. **正确性优先**  
   优先确保启动流程、地址映射、内存分配和任务切换逻辑稳定可靠。

2. **结构清晰**  
   将控制台、驱动、内存管理、调度器、中断和用户接口拆分到不同模块，便于后续扩展。

3. **便于调试**  
   保留了 QEMU、GDB 和脚本化启动方式，方便定位早期启动阶段的问题。

4. **可持续扩展**  
   预留了 syscall、userlib、syslib 等模块，为未来支持用户态程序、更多系统调用和更完整的进程模型做准备。

## 功能特性

- `no_std` / `no_main` Rust 内核
- RISC-V `Sv39` 分页机制
- 早期 MMU 初始化与高半区映射
- 基于设备树的内存探测与保留区管理
- 页表与物理页分配管理
- Buddy 分配器
- UART 串口控制台输出
- Trap / 中断处理
- 基于定时器的任务调度
- 任务创建、阻塞、唤醒与退出
- 基础系统调用接口
- 支持 QEMU `virt` 机器

## 启动流程

内核从上电到进入调度器，大致经历以下流程：

### 1. 汇编入口

项目通过 `src/entry.S` 作为入口，在最初阶段完成必要的寄存器和运行环境准备，然后跳转到 Rust 入口函数。

### 2. 早期 MMU 初始化

在 `rust_main` 中，内核会先启用早期页表映射，使得后续代码可以切换到高半区虚拟地址空间执行。这样可以把物理地址和虚拟地址的布局分离开，方便内核后续管理。

### 3. 解析设备树

内核从传入的 DTB 中读取内存信息，识别 RAM 区域、保留区域以及设备节点，为后续内存布局和 MMIO 映射做准备。

### 4. 初始化内存管理

内核会建立自身需要的页表结构，并初始化内存块管理器和 Buddy 分配器。这样可以为页面分配、任务栈分配、内核对象分配等操作提供基础能力。

### 5. 初始化中断与陷入处理

内核设置 `stvec`、`sscratch` 等寄存器，注册 trap 入口，并初始化 supervisor 级别中断逻辑，包括定时器中断相关功能。

### 6. 初始化调度器并创建任务

内核启动后会初始化调度器，创建 idle 任务以及一些测试任务，用来验证调度流程、睡眠唤醒机制和系统调用路径。

### 7. 进入调度循环

最后内核进入调度器运行状态，由时钟中断驱动任务切换，保持系统继续运行。

## 目录说明

### `src/main.rs`

内核主入口，负责启动流程、内存初始化、调度器启动，以及示例任务创建。

### `src/mm/`

内存管理相关模块，包括地址转换、页表、Buddy 分配器、Bump 分配器、内存块管理和 Slub 相关逻辑。

### `src/task/`

任务系统与调度器实现，包括任务上下文、任务控制块、上下文切换和任务状态管理。

### `src/trap/`

trap 和中断相关逻辑，包括中断初始化、trap 入口和具体服务处理。

### `src/driver/`

驱动抽象与具体设备驱动实现，当前主要包括 UART 和 PLIC 相关内容。

### `src/console/`

控制台输出支持，包括早期输出和统一打印接口。

### `src/syslib/` 与 `src/userlib/`

系统调用相关接口，分别面向内核侧和用户侧的抽象。

### `src/data_struct/`

项目中使用的数据结构与同步原语封装，例如锁、环形缓冲区和同步引用容器等。

### `src/bsp/`

板级支持包，当前主要面向 QEMU `virt` 平台。

## 依赖环境

构建和运行这个项目需要以下工具：

- Rust 工具链
- `cargo`
- `riscv64gc-unknown-none-elf` 目标
- `qemu-system-riscv64`
- `rust-gdb` 或兼容的 GDB
- `rustsbi.bin`

如果你还没有安装 RISC-V 目标，可以使用：

```text
rustup target add riscv64gc-unknown-none-elf
```

## 构建方法

使用 release 模式构建内核：

```text
cargo build --release
```

生成的内核文件通常位于：

```text
target/riscv64gc-unknown-none-elf/release/charlotte_os
```

## 运行方法

仓库提供了一个启动脚本，用于编译并在 QEMU 中运行内核：

```text
./run.sh
```

该脚本会自动完成以下工作：

- 使用 release 模式编译项目
- 启动 `qemu-system-riscv64`
- 加载 `rustsbi.bin`
- 载入内核镜像
- 以 `-nographic` 模式运行

你也可以手动启动 QEMU，命令形式类似：

```text
qemu-system-riscv64 \
    -machine virt \
    -nographic \
    -smp 1 \
    -bios rustsbi.bin \
    -kernel target/riscv64gc-unknown-none-elf/release/charlotte_os
```

## 调试方法

如果你想调试内核，可以使用调试脚本启动 GDB 会话：

```text
./debug.sh
```

常见流程如下：

1. 先以暂停状态启动 QEMU。
2. 使用 `rust-gdb` 连接目标。
3. 通过 `debug.gdb` 加载调试命令。
4. 在早期启动、内存初始化或调度器代码中设置断点并单步跟踪。

如果你想了解更细的调试启动过程，可以参考 `debug.gdb` 和 `start_qemu.sh`。
